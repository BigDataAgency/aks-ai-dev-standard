# Command: PM Requirement

## ใช้เมื่อ
ต้องการวิเคราะห์ requirement, PRD, scope, acceptance criteria ก่อนส่งต่อ dev

## Copy this into AI

```text
ทำงาน: PM Requirement
Context: <วาง business need/PRD/brief/stakeholder constraints>
โปรดทำตามขั้นตอนนี้:
1. ระบุ goal, stakeholder, constraint, deadline
2. แยก scope / out-of-scope
3. เขียน acceptance criteria ที่ dev/test ใช้ได้
4. ระบุ open questions, risk, dependency
5. ระบุ next step และ owner
6. ส่ง work event ของ command นี้ด้วย `work_type=pm-requirement`

Output ที่ต้องส่ง: requirement brief พร้อม scope/out-of-scope, acceptance criteria, open questions, risk, next step

Output ที่ต้องส่งต้องมีหัวข้อ: BDA Standard files used, Pipeline trace, Commands run, Verification / Evidence, Limitations / Risks / Next steps
```

## Checklist
- [ ] ระบุ goal/stakeholder/constraints
- [ ] แยก scope/out-of-scope
- [ ] มี acceptance criteria
- [ ] มี open questions/risk/dependency
- [ ] ส่ง work event หรือระบุเหตุผลที่ส่งไม่ได้
