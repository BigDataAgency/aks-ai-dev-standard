# คู่มือใช้ AI ทำงาน dev ให้ "เสร็จจริง" (อิงผลวัดจริง)

> who/when/why: กลั่นจากการทดสอบวัดผลจริง 2026-07-05 (Hermes vs Claude Code CLI, โจทย์จริง n≥3 ต่อ scenario, verify อิสระทุกครั้ง — ไม่เชื่อคำ AI อ้าง) โดยทีม infra/eval. อัปเดตเมื่อเทสรอบใหม่
> ใช้คู่กับ: `docs/troubleshooting-ai-access.md` (เมื่อระบบมีปัญหา) และ commands ใน repo นี้

## 3 กติกาเหล็ก (จากข้อมูลจริง ไม่ใช่ความรู้สึก)

### 1. ซอยงานเล็ก — หลักฐานชัดที่สุดของทั้งการทดสอบ
- โจทย์ใหญ่เปิดกว้าง ("สร้างทั้ง platform"): **0/8 รันที่ได้ครบ โค้ด+test ผ่าน+commit** (ทั้ง Hermes และ Claude Code)
- โจทย์เล็ก scope ชัด (1 service + tests): มีรันเขียวจริง (ดีสุด 26/26 test ผ่าน)
- → สั่ง 1 service / 1 endpoint / 1 bug ต่อครั้ง แล้วให้ test ผ่านก่อนไปชิ้นถัดไป

### 2. อย่าเชื่อคำว่า "เสร็จ" — ตรวจ 2 อย่างเสมอ
เจอจริงระหว่างทดสอบ: AI รายงาน "คอมมิตทั้งหมด 4 ไฟล์แล้ว พร้อม unit test ครอบคลุม" ทั้งที่**ไม่มีไฟล์อะไรเลยบนดิสก์**
- ✅ เห็นผล `npm test`/`pytest` **ที่รันจริง** (ไม่ใช่ AI เล่าว่าผ่าน)
- ✅ `git log` มี commit จริง
- AI ใช้คำ "เสร็จสมบูรณ์ / production-ready / ครบทุกระบบ" = สัญญาณให้ตรวจละเอียดขึ้น ไม่ใช่สัญญาณให้วางใจ

### 3. งานสำคัญรัน 2-3 รอบ — ผลแต่ละรอบไม่เหมือนกันจริงๆ
โจทย์เดียวกัน 3 รอบ: เขียว 26/26 → เขียว 12/12 → มีบั๊ก 2 เคส (Claude) / ไม่อ่านโจทย์ → โม้ → เขียว 6/6 (Hermes)
ไม่มีเครื่องมือไหนได้ 3/3 — variance คือธรรมชาติของ agentic AI ตอนนี้

## ใช้ command เสมอ อย่าสั่งลอยๆ
`/fix-bug`, `/build-feature` มี checklist บังคับ verify ในตัว — ตอนทดสอบ สั่งลอยๆ แล้ว AI ไม่อ่านโจทย์บ้าง โม้บ้าง; สั่งผ่าน command แล้วอัตราจบจริงสูงขึ้นชัดเจน

## ระวังพฤติกรรมเหล่านี้ (เจอจริงทั้งหมด)
1. **แต่งรายงานความสำเร็จ** — บอก commit แล้วทั้งที่ไม่มีไฟล์ (กติกาข้อ 2 จับได้)
2. **เขียนทับไฟล์แทน append** — งานยาวๆ AI เคยล้างไฟล์ log ที่สั่งให้ต่อท้าย (138 → 10 บรรทัด) → ไฟล์สำคัญให้ commit บ่อยๆ จะได้ diff เห็น
3. **หยุดกลางทางเงียบๆ** — งานใหญ่ AI อาจพูดว่า "ต่อไปจะทำ X" แล้วจบเลย → เช็คของจริงเสมอ
4. **ทำในหัวไม่เรียก tool** — งานเล็กมากๆ ต้องสั่ง "ใช้ tool จริง"

## Setup ที่ถูก (ย่อ — เต็มดู tool-setup + troubleshooting)
- Hermes: base URL ต้องเป็น endpoint ปัจจุบัน (`bda config-status` เช็ค) — endpoint เก่า `ai.bda.co.th` ตายแล้ว
- Claude Code CLI: `CLAUDE_CONFIG_DIR` ใหม่ + `ANTHROPIC_BASE_URL` + `ANTHROPIC_API_KEY` (แก้ 401 key เก่าค้าง)
- อย่าตั้ง max_tokens ต่ำ (<2000) — ได้คำตอบว่าง
- Claude Code session ยาวกิน token quota เร็วมาก (ส่ง context ซ้ำทุก turn) — ปิด session ที่ไม่ใช้, เจอ "หายไป/ไม่ตอบ" ให้รอ 1 นาที (429) แล้วลองใหม่
